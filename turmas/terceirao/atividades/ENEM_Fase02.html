Enem - Segunda Entrega: otimizando a segurança do sistema com dupla verificação

  1. Questão — Equipe IA

    Bolar a segunda camada do script: a camada de validação do primeiro prompt. O primeiro prompt já gerou questões do ENEM; agora precisamos de um novo prompt que receba a resposta do primeiro prompt e decida se a resposta obtida é polêmica ou não. Esse validador deve devolver apenas APROVADO ou REPROVADO.

  2. Questão — Equipe Python

    Após gerar a primeira questão, pegar o retorno da IA e enviar o segundo prompt para validação (APROVADO / REPROVADO). Se aprovado, inserir a questão no banco de questões. Se reprovado, gerar outra resposta e repetir até obter uma validação positiva — máximo de 3 retries.

  3. Questão — JSON / Endpoints

    Como ficaram os endpoints com o novo JWT de usuários implementado? Relatar se existe algum endpoint que não funciona no sistema atual e descrever quais e por quê.

  4. Questão — Banco de Dados (tabelas)

    Atualizar as novas tabelas do banco de dados. Responder com a lista de tabelas geradas e fornecer exemplos de valores validados para cada tabela.

  5. Observações / Requisitos

    - O validador (segundo prompt) deve ser idempotente e retornar apenas "APROVADO" ou "REPROVADO".
    - Limite de tentativas para geração/validação: 3.
    - Em caso de "REPROVADO" após 3 tentativas, registrar a ocorrência para revisão manual.
