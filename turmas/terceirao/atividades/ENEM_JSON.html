1. Tarefa 1 — Criar JSON para nova questão

    Antes de criar um JSON para enviar ao backend, converse com a equipe Python para confirmar quais campos são obrigatórios.

    Exemplo de JSON:

    {
  "enunciado": "Qual é a capital do Brasil?",
  "alternativas": [
    {"id":"A","texto":"São Paulo"},
    {"id":"B","texto":"Brasília"},
    {"id":"C","texto":"Rio de Janeiro"},
    {"id":"D","texto":"Salvador"}
  ],
  "resposta_correta": "B",
  "nivel_dificuldade": "facil",
  "competencia": "Geografia"
}

    Dicas:

    • Valide com Python se todos os campos estão corretos.
    • Confirme com o banco de dados se os nomes das colunas batem com o JSON.
    • Mantenha os IDs das alternativas consistentes.

  2. Tarefa 2 — JSON de listagem de questões

    Antes de consumir a lista de questões, converse com a equipe Python sobre como o backend vai devolver os dados. 

    {
  "page": 1,
  "per_page": 10,
  "total": 42,
  "items": [
    {
      "id": 1,
      "enunciado": "Qual é a capital do Brasil?",
      "nivel_dificuldade": "facil",
      "upvotes": 3,
      "created_at": "2025-09-01T12:34:56Z"
    }
  ]
}

    Confirme com Python se campos como page, total e items estão corretos antes de usar no frontend.

  3. Tarefa 3 — JSON de upvote

    Ao atualizar o número de upvotes, converse com Python para garantir que o JSON retornado esteja correto. 

    {
  "id": 1,
  "upvotes": 4
}

    A equipe JSON deve validar com Python que o campo upvotes reflete o novo valor no banco de dados.

  4. Tarefa 4 — Teste de comunicação

    Para garantir que tudo funciona, teste o JSON com a equipe Python e confirme que os dados chegam corretamente no banco. 

    {
  "enunciado": "Qual é o maior estado do Brasil?",
  "alternativas": [
    {"id":"A","texto":"Amazonas"},
    {"id":"B","texto":"São Paulo"},
    {"id":"C","texto":"Minas Gerais"},
    {"id":"D","texto":"Bahia"}
  ],
  "resposta_correta": "A",
  "nivel_dificuldade": "medio",
  "competencia": "Geografia"
}

    Envie este JSON para /api/questions e valide com Python e banco de dados se foi salvo corretamente.

Resumo: Comunicação é essencial

  • Antes de criar qualquer JSON, confirme campos e estrutura com a equipe Python.
  • Valide nomes de colunas e tipos de dados com o time de Banco de Dados.
  • Teste cada JSON em conjunto com as equipes para evitar erros de integração.
  • Use exemplos pequenos para facilitar conferência e entendimento.

Exemplos de requisições

1️⃣ Listar questões (GET /api/questions)

Lista as primeiras 10 questões e exemplo de busca:

# Lista as primeiras 10 questões
curl -X GET "https://gsuzeda.pythonanywhere.com/api/questions?page=1&per_page=10" \
     -H "Content-Type: application/json"

# Buscar questões com texto específico
curl -X GET "https://gsuzeda.pythonanywhere.com/api/questions?page=1&per_page=10&search=capital" \
     -H "Content-Type: application/json"

2️⃣ Inserir nova questão (POST /api/questions)

Exemplo de payload para criar uma nova questão:

curl -X POST "https://gsuzeda.pythonanywhere.com/api/questions" \
     -H "Content-Type: application/json" \
     -d '{
           "enunciado": "Qual é a capital do Brasil?",
           "alternativas": [
             {"id":"A","texto":"São Paulo"},
             {"id":"B","texto":"Brasília"},
             {"id":"C","texto":"Rio de Janeiro"},
             {"id":"D","texto":"Salvador"}
           ],
           "resposta_correta": "B",
           "nivel_dificuldade": "facil",
           "competencia": "Geografia"
         }'

A resposta deve retornar o id da nova questão e a mensagem de sucesso.

3️⃣ Dar upvote em uma questão (POST /api/questions/{id}/upvote)

Exemplo de requisição para incrementar o contador de upvotes:

# Supondo que o id da questão seja 1
curl -X POST "https://gsuzeda.pythonanywhere.com/api/questions/1/upvote" \
     -H "Content-Type: application/json"

A resposta mostrará o id da questão e o novo valor de upvotes.

4️⃣ Listar usuários (GET /usuarios)

curl -X GET "https://gsuzeda.pythonanywhere.com/usuarios" \
     -H "Content-Type: application/json"

5️⃣ Processar JSON simples (POST /processar)

curl -X POST "https://gsuzeda.pythonanywhere.com/processar" \
     -H "Content-Type: application/json" \
     -d '{"nome": "Maria"}'

Retorna uma mensagem de confirmação e o JSON enviado.
